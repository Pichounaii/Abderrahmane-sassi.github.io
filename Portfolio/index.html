Architecture réseau sécurisée avec Split-Brain DNS

pfSense • DNS avancé • VPN • Cisco routing

Architecture sécurisée : Split-Brain DNS, segmentation réseau, accès distant VPN

🎯 Contexte & Objectif

Mise en place d'une architecture réseau avancée intégrant un système DNS Split-Brain pour différencier les résolutions internes et externes. Objectif : sécuriser l'accès aux services, minimiser l'exposition de l'infrastructure interne et garantir une continuité de service en cas de panne.

🔍 Problématique technique

Dans une architecture classique, le DNS public expose la topologie interne (noms de serveurs, services). Les utilisateurs internes interrogent inutilement le DNS public pour accéder à des ressources locales, générant latence et trafic sortant. En cas de coupure Internet, les services internes deviennent inaccessibles malgré leur disponibilité locale.

⚙️ Solution mise en œuvre

🔵 Split-Brain DNS
  • Serveur primaire (192.168.10.10) — Zone interne technoservice.local, enregistrements privés
  • Serveur secondaire (192.168.10.20) — Failover automatique, réplication zone
  • Zone externe (technoservices.fr) — Enregistrements publics limités, masquage infra interne
  • Redirection conditionnelle — Requêtes externes → DNS publics (8.8.8.8), internes → serveurs locaux
🔴 Sécurité réseau
  • pfSense firewall — Multi-WAN (Fibre + 4G), règles ACL strictes, NAT, port forwarding minimal
  • VPN WireGuard — Accès distant sécurisé pour télétravail, tunnel chiffré, authentification par clés
  • Isolation des flux — VLAN dédiés, pas de routage direct entre zones
🟢 Infrastructure
  • Routeur Cisco C3560 — Routage inter-VLAN, trunk 802.1q, SVI par segment
  • OpenLDAP (Ubuntu) — Annuaire centralisé, authentification services
  • Client Windows 10 — Tests intégration domaine, validation résolution DNS

🔧 Réalisation technique

  • Configuration DNS Split-Brain — Création de deux vues DNS (interne/externe) avec zones distinctes, enregistrements A/CNAME différenciés, tests de résolution depuis différents points du réseau
  • Haute disponibilité DNS — Configuration maître-esclave entre serveurs, réplication de zone, tests de basculement automatique, monitoring disponibilité (Zabbix)
  • Sécurisation firewall — Règles deny-all par défaut, whitelist services essentiels, logging connexions suspectes, limitation tentatives bruteforce
  • VPN WireGuard — Génération paires de clés, configuration serveur pfSense, création profils clients, tests connexion distante, validation accès ressources internes
  • Routage inter-VLAN sécurisé — Configuration SVI Cisco, ACL inter-VLAN, validation isolation (tests ping, scan ports), documentation flux autorisés

💡 Compétences démontrées

DNS avancé (Split-Brain) Haute disponibilité pfSense firewall VPN WireGuard Routage Cisco Segmentation réseau Analyse de sécurité Documentation technique

📈 Résultat & Impact

Résolution DNS interne 10x plus rapide (cache local). Exposition externe réduite de 80% (uniquement services publics nécessaires). Continuité de service garantie même en cas de panne Internet (services internes accessibles). Accès VPN sécurisé pour télétravail sans compromis sur la sécurité.

Enseignement clé : La sécurité commence par la segmentation et l'isolation. Ne jamais exposer plus que le strict nécessaire. Un DNS bien configuré est un outil de sécurité à part entière.
